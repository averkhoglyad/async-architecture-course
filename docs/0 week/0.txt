Пользовательское приложение при логине получает токен из SSO-сервиса, в дальнейшем все запросы к беку идут с этим токеном
Для работы с каждой подсистемой (Таск-Трекер, Аккаунтинг, Аналитика) выделен отдельный сервис. Сервисы при необходимости получают данные о Пользователях обращаясь к SSO-сервису. При Создании/Редактировании/Ассайне задач сервис Таск-трекер сохраняет детали задачи в БД и посылает запрос в очередь событий для обработки этих задач - начислении списании с баланса, запись событий аудита и т.п. Сервис Аккаунтинга получает данные по балансу из БД, дополнительных требований и функционала у него вроде нет. Сервис Аналитики запрашивает данные из БД, возможно потребуются даные из разных баз
